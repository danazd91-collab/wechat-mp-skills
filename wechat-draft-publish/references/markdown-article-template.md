# Markdown 发文模板

下面这份模板适合直接用于公众号长文写作，并配合 `minimal / official / khazix / bijixia` 主题发布到公众号草稿箱。

建议原则：

- 用 Markdown 写结构，不在正文里硬塞样式
- 导语、说明块、金句分别交给 `intro / card / quote`
- 小标题尽量明确，方便长文阅读和主题渲染

```md
# 文章标题

:::intro
导语 / 编者按 / 开头一句话总结。

这里用 1 到 2 段把文章的主题先说清楚。
:::

开场第一段。

开场第二段。

**一句你想单独强调的话。**

## 第一部分标题

这一部分的正文第一段。

这一部分的正文第二段。

:::quote
一句你想单独拎出来的重点话。

也可以是两段式金句。
:::

这一部分的正文第三段。

![这里填写图片说明](这里填写图片地址)

## 第二部分标题

这一部分的正文第一段。

这一部分的正文第二段。

:::card
这里适合放：

- 背景说明
- 主办方简介
- 方法总结
- 补充信息
:::

这一部分的正文第三段。

- 要点一
- 要点二
- 要点三

## 第三部分标题

这一部分的正文第一段。

这一部分的正文第二段。

---

收尾正文第一段。

收尾正文第二段。

:::quote
结尾金句 / 你想单独落下来的那句话。
:::
```

## 最小版本

如果你不想一开始用太多结构，至少保留这几个：

```md
# 标题

:::intro
导语
:::

## 小标题

正文

:::quote
重点句
:::
```

## 使用建议

- `intro`：只在开头用，避免全篇出现太多导语块
- `card`：只装说明性内容，不要把普通正文也塞进去
- `quote`：适合金句、观点总结、情绪落点
- 图片说明尽量短，避免图注比图片更抢戏
- 如果一篇文章完全没有 `##`，长文阅读压力会明显变大
